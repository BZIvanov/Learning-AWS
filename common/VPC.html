VPC - Virtual Private Cloud

VPC is private network where we can deploy our resources

CIDR - Classless Inter-Domain Routing

It is a method for efficiently allocating and specifying IP addresses and routing them on the Internet. It is notation to specify IP address ranges for virtual networks, subnets, and other resources.

For example, if you have an IP address range starting at 192.168.0.0 and want to allocate 256 addresses, the CIDR notation would be 192.168.0.0/24. Here, the "/24" indicates that the first 24 bits of the IP address are fixed as the network address, leaving 8 bits for host addresses (2^8 = 256).

Subnets

Subnets allow us to partition our network inside the VPC

Public subnet is a subnet which is accessible from internet

Private subnet is a subnet which is not accessible from internet

Access from internet and between subnets is defined in Route Tables

Internet Gateway and NAT Gateways

Internet Gateways provide us a way to access our VPC from internet. Public subnets have a route to the internet gateway

NAT Gateways and NAT Instances provide us a way our Private Subnets to access internet while remaining private.

VPC Peering

Connect two VPC privately using AWS's network. It makes them to behave like they are the same network.

VPC Endpoints

VPC Endpoints provide private access to AWS service (for example S3, SNS, etc...) within VPC so we don't have to use the public internet.

Endpoint types

  • Interface endpoints
  • Gateway endpoints
