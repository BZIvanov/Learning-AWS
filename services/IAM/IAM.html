IAM - Identity and Access Managment

Main points in IAM are:

  • Users - usually physical person
  • Groups - Could be admins, devops, etc... Groups only contain users, not another groups. One user can be in more than 1 group.
  • Policies - they define what each of the 2 above can do. Can be Allow or Deny
  • Roles - internal usage with AWS resources. Also collection of policies, for example ("SNS:CreateTopic", "SNS:Subscribe"). Roles are similar to policies, they define what services are allowed to do similarly like what users are allowed to do by policies. Common roles are EC2 instance roles, Lambda function roles, with the roles this services are allowed to do some actions.

MFA setup

When setting up MFA device we can choose from the following options

  • Virtual MFA device
  • U2F Security key
  • Other hardware MFA device

IAM Security tools

  • IAM credentials report (account level) - report that lists all account users and their credentials status
  • IAM access advisor (user level) - shows granted permissions and when they were last accessed
