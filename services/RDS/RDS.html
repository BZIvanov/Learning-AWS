RDS - Relational Database Service

Big advantage of using RDS is the storage auto scaling. If we are running out of storage it will be increased automatically.

Data recovery types

  • Automated backups
  • Snapshots - they are done manually and are still kept after we delete the RDS instance.

Multi AZ RDS

Multi AZ allows us to have the exact copy of our DB in another AZ. In case of some disaster automatically to copy will be used.

The intention of Multi AZ is for disaster recovery, not performance improvements. For performance improvements we would use Read Replicas.

Read Replica

RR allows us to have read-only copy of our production database. This is achieved by using async replication from our primary RDS instance to the read replica.

Read Replicas can be promoted to be a database. This breaks the replication.

Each RR will have its own DNS endpoint.

We can have up to 5 RRs. They can be within the same AZ, Cross AZ or Cross Region.

Encryption

At rest encryption

Encryption has to be defined at launch time. If the master is not encrypted the read replicas also cannot be encrypted.

In-flight encryption

Encryption is done with SSL certificates.
