SQS - Simple Queue Service

SQS is good for decoupling applications. The sender to the queue is the Producer (first app) and receiver from the queue is the Consumer (second app).

Default retention period (the time we store the message in the queue) is 4 days with optional maximum of 14 days.

Access policy

Similarly to S3, SQS also have its own access policies.

  • Cross account access
  • Publish S3 event notifications to SQS

Message visibility timeout

After a message is polled by a consumer, it becomes invisible to other consumers.

By default the message is invisible for 30 seconds, which means that we have to process it in 30 seconds.

Dead letter queue (DLQ)

If the consumer fails to process the message and 30 seconds of invibility expired the message will be returned to the queue (become visible again). With DLQ we can control how many times we will return the message back to queue. This is useful, because if we keep failing with processing the message, there is the chance that something is wrong with the message and we should stop trying.
