Route53

Route53 is managed DNS (Domain Name System)

DNS is collection of rules and records which help clients to understand how to reach a server through its domain name.

Record Types

  • A - hostname to IPv4
  • AAAA - hostname to IPv6
  • CNAME - hostname to hostname
  • NS - name servers for the hosted zone
  • Alias - hostname to an AWS resource

Routing policies

  • Simple routing - route traffic to a single resource
  • Weighted - control percent of the requests going to specific AWS resource. Different wights are assigned to different records.
  • Geolocation - based on user location. For example we can redirect users from Spain to one IP and users from Italy to another IP. We can also specify Default case for all the others.
  • Latency - redirects to resource, that has the least latency close to us
  • Failover - it is based on Health checks
  • Multivalue answer - routing to multiple resources
