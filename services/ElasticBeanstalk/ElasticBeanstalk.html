ElasticBeanstalk

ElasticBeanstalk is about deploying our applications on AWS. ElasticBeanstalk is managed service and in general what it does for us is automatically handles capacity provisioning, load balancing, scaling, health monitoring, instance configuration, etc...

Deployment Modes

  • Single instance - good for development
  • High availability with Load Balancer - good for production

Deployment options for updates

  • All at once - fastest option, but instances are not available while the downtime for update
  • Rolling update - update few instances at a time and then move to the next bucket once the first one is healthy
  • Rolling with additional batches - like rolling, but spins up new instances to move the batch (this way the old application is still available)
  • Immutable - spins up new instances in a new ASG, deploys version to these instances and then swaps all the instances when everything is healthy

Lifecycle Policy

ElasticBeanstalk can store at most 1000 application versions.

We can phase out (remove) versions by using lifecycle policy. They can be 2 types:

  • based on time - old versions are removed
  • based on space - when we have too many versions
